# 09 — Dataset

## 9.1 Dataset creation pipeline

```
Elasticsearch (honeypot-logs-*)
   │  01_ingest.py scrolls all events
   ▼
data/raw.parquet (~2k rows × ~25 cols)
   │  02_features.py engineers features + scales + one-hot encodes
   ▼
data/features.parquet (~2k rows × ~40 cols)
   │  also exported as CSV for sharing
   ▼
datasets/honeypot_dataset.csv (labeled subset, ~18 cols)
```

The ML target is `attack_class` (11 classes).

## 9.3 Labeling methodology

Labels come from two layers:

1. **Per-container heuristic** (`attack_type_hint`): a cheap regex-based
   classifier inside each `app.py`. This produces a *weak* label.
2. **Logstash mapping** (`attack_class`): a deterministic map from
   `attack_type_hint` to one of 11 coarse classes. This produces the *strong*
   label used by the ML pipeline.

Because every event is generated by a script with a known intent (every curl
in `03_webattacks.sh` is a deliberate attack), the labels are effectively
ground truth for the lab.

## 9.4 Class distribution (typical run)

For a single execution of `run_all_attacks.sh` + 30 rounds of seed traffic:

| `attack_class` | Count | %      |
|-----------------|-------|--------|
| benign          | ~600  | ~30%   |
| recon           | ~800  | ~40%   |
| bruteforce      | ~540  | ~27%   |
| sqli            | ~25   | ~1%    |
| cmdi            | ~25   | ~1%    |
| xss             | ~5    | <1%    |
| traversal       | ~5    | <1%    |
| exfil           | ~2    | <1%    |
| implant         | ~3    | <1%    |
| ssrf            | ~3    | <1%    |
| abuse           | ~2    | <1%    |

The dataset is **imbalanced** — benign/recon/bruteforce dominate. We address
this in training via `class_weight="balanced"` and macro-averaged F1.

## 9.5 Generating a larger / more balanced dataset

To balance the classes:

1. Run `09_custom_payloads.sh` 10× to multiply the rare classes.
2. Run `08_dos_sim.sh` with `N=2000` for more benign-clustered-in-time events.
3. Run the seed-traffic script for 10 minutes continuously.
4. Optional: apply SMOTE (`imbalanced-learn`) on the minority classes in
   `02_features.py`.

## 9.7 Data privacy

All `src_ip` values in the lab dataset are private IPs (`192.168.1.30`,
`172.20.0.1` for seed traffic). No PII is collected. The dataset is safe to
share with evaluators and to include in the project report.

## 9.8 Reusing the dataset

The CSV in `datasets/honeypot_dataset.csv` can be loaded in any tool:

```python
import pandas as pd
df = pd.read_csv("datasets/honeypot_dataset.csv")
print(df["attack_class"].value_counts())
```

It is also suitable for independent re-training in other frameworks
(TensorFlow, PyTorch, Weka, RapidMiner).
